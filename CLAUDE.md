# NewMikdash Font Project

Hebrew font family reverse-engineered from a Dead Sea Scrolls scholarly transcription, with custom diacritical marks (diamond РЌє and circle РЌІ above letters).

## Two Weights

- **NewMikdash-Bold** Рђћ filled (solid) Hebrew glyphs
- **NewMikdash-Regular** Рђћ hollow (outline) Hebrew glyphs, generated by inward polygon offset via pyclipper

## Quick Start

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
brew install potrace  # needed for bitmapРєњSVG vectorization

# Build from existing font (recommended, faster):
python -m scripts.assemble_from_font

# Build from scanned image (full pipeline):
python scripts/build.py
```

Output: `output/NewMikdash-Bold.ttf` and `output/NewMikdash-Regular.ttf`

## Two Build Paths

### Path A: From Existing Font (`scripts/assemble_from_font.py`)

Extracts Hebrew glyphs from `FrankRuehlCLM-Medium.otf`, adds marks, builds both weights. This is the active/primary path. Uses pre-composed PUA glyphs for marks (see below).

```bash
python -m scripts.assemble_from_font
```

Default source: `FrankRuehlCLM-Medium.otf`, donor for Latin: `donor/EBGaramond-Bold.ttf`

### Path B: From Scanned Image (`scripts/build.py`)

Full 4-stage pipeline: preprocess Рєњ segment Рєњ vectorize Рєњ assemble. Uses `scripts/assemble.py` (not `assemble_from_font.py`). This path still uses GPOS mark-to-base positioning, which does NOT work in browsers for PUA codepoints (see "Critical Lesson" below).

```bash
python scripts/build.py --source font.jpg --donor donor/EBGaramond-Bold.ttf
```

**Note:** Path B's mark positioning is broken in browsers. If you need marks to work in browsers, use Path A or port the pre-composed approach to Path B's `assemble.py`.

## Mark Positioning: What Works and What Doesn't

### THE CRITICAL LESSON

**Browsers (Chrome, Safari, Firefox) will NOT apply GPOS or GSUB OpenType features across a Hebrew letter (U+05D0РђЊ05EA) and a Private Use Area character (U+E000+).** This is because the HarfBuzz text shaper splits them into separate "shaping runs" based on Unicode script assignment. PUA characters have script=Common, Hebrew letters have script=Hebrew, and the shaper processes them independently.

We tried ALL of the following Рђћ none work in browsers:

| Approach | Why it fails |
|----------|-------------|
| GPOS mark-to-base (PUA mark + Hebrew base) | Separate shaping runs Рђћ GPOS never sees them together |
| GSUB ligature substitution (liga, ccmp, calt) | Same shaping-run split Рђћ GSUB rule never fires |
| Generic combining marks (U+0302, U+030A) | Wrong script run Рђћ Latin combining marks not shaped with Hebrew base |

**All of these work in HarfBuzz CLI and FreeType/Pillow rendering** Рђћ the problem is browser-specific shaping behavior.

### THE APPROACH THAT WORKS: Hijacked Hebrew Combining Marks

We override two real Hebrew codepoints with our custom mark glyphs:
- **U+05C4** (HEBREW MARK UPPER DOT) Рєњ diamond above
- **U+05AF** (HEBREW MARK MASORA CIRCLE) Рєњ circle above

Because these are real Hebrew-script codepoints, the browser's shaper keeps them in the **same shaping run** as Hebrew letters. Our GPOS mark-to-base anchors then position them correctly Рђћ centered above each letter based on that letter's advance width.

Usage: type a Hebrew letter followed by U+05C4 (diamond) or U+05AF (circle). The mark auto-centers.

This is implemented in `scripts/assemble_from_font.py`:
- `add_mark_glyphs()` Рђћ creates the mark glyphs
- `add_gpos_marks()` Рђћ adds GPOS mark-to-base positioning

### FALLBACK: Pre-composed PUA Glyphs

Each letter+mark combination also exists as a **single pre-composed glyph** at its own PUA codepoint (U+E100РђЊE135). The mark contours are baked into the glyph. No GPOS needed Рђћ the browser just renders one glyph. Use these when GPOS is unavailable.

### How Mark Positioning Is Calculated

```python
# Find typical glyph top (75th percentile of all glyph tops)
typical_top = glyph_tops[len(glyph_tops) * 3 // 4]
mark_y = typical_top + 150  # gap above letters

# GPOS anchors: base anchor at (advW/2, mark_y), mark anchor at (0, mark_y)
# Pre-composed: mark contours shifted to (advW/2, mark_y)
# Exception: lamed's circle mark is at 60% of advW (shifted right to avoid ascender)
```

### LSB Must Match xMin

**Critical bug we hit:** `FontBuilder.setupHorizontalMetrics()` takes `(advanceWidth, lsb)`. If LSB=0 but the glyph's contours start at x=171, the browser renders the glyph shifted left. After `setupGlyf()`, you MUST read each glyph's `xMin` and set LSB to match:

```python
fb.setupGlyf(glyph_table)
for name in glyph_names:
    g = glyph_table[name]
    if hasattr(g, 'xMin'):
        metrics[name] = (metrics[name][0], g.xMin)
fb.setupHorizontalMetrics(metrics)
```

### Side-Bearing Normalization

The source font (FrankRuehlCLM) has inconsistent side bearings Рђћ narrow letters like vav had disproportionately large gaps (30% whitespace vs 10% for wide letters). `normalize_side_bearings()` in `assemble_from_font.py` strips the source bearings and applies a consistent **6% of ink width** on each side. This runs after glyph extraction and before mark creation.

## Key Source Files

### `scripts/helpers.py`
- `HEBREW_LETTERS` Рђћ dict mapping 27 English glyph names Рєњ Unicode codepoints
- `COMPOSED_DIAMOND` / `COMPOSED_CIRCLE` Рђћ dicts mapping letter names Рєњ PUA codepoints
- `_LETTER_ORDER` Рђћ canonical order of the 27 letters (determines PUA codepoint assignment)
- `parse_svg_to_contours()` Рђћ SVG path Рєњ contour lists
- `normalize_contours()` Рђћ scale/flip contours to font coordinate space
- `compute_global_scale()` Рђћ uniform scale factor preserving relative letter sizes

### `scripts/assemble_from_font.py` (Path A Рђћ the working path)
- `extract_glyph_contours()` Рђћ extract Hebrew glyphs from source font via RecordingPen
- `add_mark_glyphs()` Рђћ create pre-composed letter+mark PUA glyphs
- `build_font_from_contours()` Рђћ build TTF with correct LSB handling
- `make_hollow_contours()` Рђћ pyclipper inward offset for Regular weight
- `build_from_source()` Рђћ main entry point

### `scripts/assemble.py` (Path B Рђћ SVG pipeline)
- Same structure but uses GPOS `mark` feature (works in CLI, NOT in browsers)
- `add_gpos_marks()` Рђћ builds feaLib GPOS mark-to-base (broken in browsers for PUA)

## Browser Cache Busting

When iterating on font builds, browsers aggressively cache fonts. To force reload:
- Change the `@font-face` `src` URL with a query param: `url('output/NewMikdash-Bold.ttf?v=2')`
- Or change the `font-family` name temporarily
- Hard-refresh alone is often not enough
